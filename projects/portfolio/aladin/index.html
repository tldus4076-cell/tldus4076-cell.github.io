현업 요청을 듣고 자동화 가능한 단위로 쪼개 빠르게 시연 가능한 결과물을 만드는 역량을 보여줍니다.

01. Problem

현업 반복 업무를 자동화 대상으로 구조화

주문 확인

운영자가 CSV 또는 엑셀로 받은 주문 데이터를 매일 확인하고 누락/오류를 찾는 반복 업무가 발생합니다.

납품 상태 분류

B2B 납품 건은 출고 가능, 확인 필요, 보류 같은 상태 분류가 필요하며 담당자 간 공유가 중요합니다.

정산 계산

주문 금액, 배송비, 수수료, 예외 건을 수기로 정리하면 시간이 오래 걸리고 실수가 생깁니다.

02. Solution

작게 만들고 빠르게 검증하는 자동화 흐름

이 프로토타입은 CSV 데이터를 읽었다고 가정하고 주문 상태를 자동 분류합니다. 이후 정산 합계, 예외 건, 담당자 공유 메시지를 자동 생성하여 운영자가 바로 확인할 수 있게 합니다.

  1. 현업 담당자 인터뷰로 반복 업무와 판단 기준 정리
  2. CSV 컬럼과 상태 분류 규칙 정의
  3. 주문별 출고 가능 여부와 예외 사유 자동 판정
  4. 정산 금액과 공유 메시지 자동 생성
  5. 프로토타입 검증 후 API/내부 시스템 연동 범위 분리

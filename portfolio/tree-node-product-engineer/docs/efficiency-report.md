## 3. 메모리 vs 속도 트레이드오프

| 구조 | 메모리 | 속도 | 용량 |
|------|--------|------|------|
| 동기 직접 호출 | 적음 | 느려지음 | 100명 이하 |
| 비동기 + 캐시 | 약간 더 사용 | 빠름 | 100~500명 |
| Redis + 영소저지 | 점잠 사용 | 가장 빠름 | 500명+ (프로덕션) |

## 4. 충동을 준 경험

1. 처음엔 dict의 기본 get/set으로 마무이 끌여서 하나씩 처리했음
2. 다중 사용자 동시 접속 시 다른 사람의 자원이 남의 데로 실수패시켰음
3. 대안: 동기 직접 호출 → 이벤트 버스 + 자원 바인드링 + 락 지역 조절
